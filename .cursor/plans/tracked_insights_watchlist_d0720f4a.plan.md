# Tracked Insights Watchlist

## Current State

- Insights are ephemeral: each refresh deletes all insights for that `date_filter` + `channel_group` and regenerates
- No versioning, no history, no tracking over time
- `generated_insights` table stores current batch only
- No post-sync hooks exist -- sync just updates `last_synced_at` and writes to `los_sync_history`

## Post-Sync Hook System (Foundation)

Central to tracked insights and future features (auto-refresh insights, cache invalidation, etc.).

- **New file**: `server/src/services/hooks/postSyncHookService.ts`
  - Registry pattern: `registerHook(name, fn)`, `runPostSyncHooks(context)`
  - Context: `{ tenantId, connectionId, syncType, recordsSynced, loansAdded, loansUpdated }`
  - Hooks run async (don't block sync completion)
- **Integration points** (insert after successful status update):
  - `[server/src/services/etl/encompassEtlService.ts](server/src/services/etl/encompassEtlService.ts)` -- after line ~184 (status update)
  - `losApiService` -- after status update
  - `csvProcessor` -- after status update
- Initial hook: tracked insight evaluation. Future: insight regeneration, cache busting.

## New Database Tables (Tenant DB Migration)

`**tracked_insights**` -- the watchlist:

- `id` (UUID PK)
- `user_id` (UUID), `user_email` (TEXT)
- `headline` (TEXT) -- original insight headline
- `understory` (TEXT) -- original understory
- `metric_signature` (JSONB) -- SQL query + conditions that define what this insight tracks (extracted from insight's `evidence_table` SQL or generated by agent)
- `source_insight_id` (INT, nullable) -- reference to original `generated_insights` row
- `source_type` (TEXT) -- `'pipeline'` | `'research'` | `'manual'`
- `status` (TEXT) -- `'active'` | `'resolved'` | `'archived'`
- `alert_threshold` (JSONB, nullable) -- conditions for alerting (e.g., metric change > 20%)
- `tags` (TEXT[])
- `created_at`, `updated_at`

`**tracked_insight_snapshots**` -- time-series evaluations:

- `id` (UUID PK)
- `tracked_insight_id` (UUID FK)
- `metric_values` (JSONB) -- e.g., `{ "pipeline_risk_pct": 23.5, "loans_at_risk": 47 }`
- `previous_values` (JSONB, nullable)
- `change_summary` (TEXT) -- LLM-generated one-line summary of what changed
- `trend` (TEXT) -- `'improving'` | `'worsening'` | `'stable'` | `'new'`
- `evaluated_at` (TIMESTAMPTZ)

## Tracked Insight Evaluator Service

**New file**: `server/src/services/insights/trackedInsightEvaluator.ts`

- `evaluateTrackedInsights(tenantId, tenantPool)`:
  - Loads all `status = 'active'` tracked insights for the tenant
  - For each insight:
    - Executes `metric_signature` SQL against current data via `safeExecuteSQL` from `[server/src/services/research/tools.ts](server/src/services/research/tools.ts)`
    - Fetches previous snapshot for comparison
    - Determines `trend` (improving/worsening/stable) via simple numeric comparison or lightweight LLM call for complex metrics
    - Generates `change_summary` (rule-based for simple deltas, LLM for nuanced changes)
    - Stores new row in `tracked_insight_snapshots`
    - If `alert_threshold` is met, log it (future: in-app notifications)

## API Endpoints

**New route**: `server/src/routes/trackedInsights.ts`, registered at `/api/insights/tracked`

- `POST /` -- pin an insight (accepts headline, understory, metric_signature, source_insight_id)
- `GET /` -- list user's tracked insights with latest snapshot and trend
- `GET /:id/history` -- get time-series snapshots for sparkline rendering
- `PUT /:id` -- update status, alert_threshold, tags
- `DELETE /:id` -- untrack/remove

## Wiring

Register the evaluator as a post-sync hook so it runs after each 4-hour data load cycle:

```
registerHook('tracked-insight-evaluation', async (ctx) => {
  await evaluateTrackedInsights(ctx.tenantId, ctx.tenantPool);
});
```
