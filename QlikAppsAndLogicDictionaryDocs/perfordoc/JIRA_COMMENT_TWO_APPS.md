# Jira Comment - Two-App Approach Decision

[~mark.roszko] After assessing the complexity of combining Retail and TPO data and functionality into one performance app, it was determined that having 2 separate performance apps is the best path forward. The logic for having either Retail or TPO but not both is already in place in the loadscripts/frontend, so this approach leverages existing infrastructure. Clients that have TPO data will see a second performance app with TPO-only data. Each app will have simplified logic with fixed actor types, eliminating the need for dynamic channel and actor selection, complex show conditions, actions, and conditional expressions. This approach provides better performance with smaller data models and faster reloads, while also making both apps easier to maintain and providing a clearer user experience where users know which app to use based on their role.
